# Supertrip

Mobile PWA for the Eby family Japan/Korea trip, July 22 – Aug 5, 2026.

## Files

| File | What it is |
|---|---|
| `index.html`    | The actual PWA (what runs on phones) |
| `desktop.html`  | iPhone-15 Pro framed preview for design review (default on desktop browsers) |
| `styles.css`    | All styles |
| `app.js`        | App logic (single-file, vanilla JS) |
| `data.js`       | Trip data — **auto-generated by `sync.mjs`** |
| `sw.js`         | Service worker (offline tiles + app shell) |
| `sync.mjs`      | Pulls latest data from Coda, regenerates `data.js` |

## Family setup (one-time, per phone)

1. Send everyone the deployed URL: `https://supertrip-mobile.vercel.app`.
2. They open it in Safari → Share → **Add to Home Screen**.
3. The icon now lives on their home screen and opens like a real app.
4. On first launch with wifi, go to **Map → ⚙ Settings → Download for offline**
   to cache map tiles. The trip data is already cached automatically.

## Updating the trip (you, when something changes)

1. Edit anything in the Coda doc — itinerary, notes, activities, todos, etc.
2. Locally:
   ```bash
   node sync.mjs
   git commit -am "sync from coda"
   git push
   ```
3. Vercel/Netlify/GitHub Pages picks up the push and redeploys automatically.
4. Next time anyone opens the app, a small **"New trip updates available · Refresh ↻"**
   banner appears in the bottom-right. Tapping it reloads with the new data.

## sync.mjs setup (one-time)

1. Generate a Coda API token at <https://coda.io/account> → API tokens
   (read-only access to the trip doc is enough).
2. Create `.env` in the project root:
   ```
   CODA_TOKEN=your_token_here
   ```
   (`.env` is gitignored — don't commit your token.)
3. Install dotenv: `npm install dotenv`
4. Run: `node sync.mjs`

The script never writes back to Coda. It only reads.

## What sync.mjs pulls from Coda vs. keeps in code

**From Coda (live, edit anytime):**
- Day names, dates, locations, notes
- Hero image URLs (per-day photo for the Today hero)
- Activity names, descriptions, times, categories, URLs
- To-do items, priorities, links, recommendations
- Flight details

**Kept in `sync.mjs` (hardcoded, edit if needed):**
- Per-day color accent (DAY_META)
- Country code per day (JP/KR)
- Day-center lat/lng for map fallback
- Per-activity lat/lng (ACTIVITY_LATLNG)

To add a brand-new activity in Coda, also add an entry to `ACTIVITY_LATLNG`
in `sync.mjs` (or leave it out — the activity will still show, just without
a map pin until you add coordinates).

## Hosting recommendation

**Vercel** is the simplest. Free for personal use:

1. Push this folder to a new GitHub repo.
2. <https://vercel.com> → Import the repo → click Deploy.
3. Production URL: `https://supertrip-mobile.vercel.app` (Vercel project name: `supertrip-mobile`).
4. Share it with the family.

GitHub Pages and Netlify also work — anything that serves static files over HTTPS.

## Offline behavior

- App shell (HTML/CSS/JS) and trip data are cached on first launch — work offline forever.
- Map tiles cache as the user pans, OR all-at-once via Settings → Download for offline.
- Hero images cache on first view.
- Weather is the only thing that requires connectivity — fails silently when offline.

## App icon

Add-to-Home-Screen uses the inline samurai-helmet SVG icon embedded in
`index.html`. To change it, edit the `apple-touch-icon` link in the `<head>`.
